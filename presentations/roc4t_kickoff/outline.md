# Roc4t — AI-Assisted Design Pipeline
## Kick-off Presentation Outline

### Slide 1 — Cover
- Title: Roc4t — AI-Assisted Design Pipeline
- Subtitle: Internal Kick-off & 30-Day Roadmap
- Visual: Dark architectural hero with gradient overlay

### Slide 2 — Context & Objective
- Context: Two remote designers for AutoCAD and 3D work. Goal: build an AI agent that performs the same tasks — better and faster.
- Objective: Unified production chain around AutoCAD, Revit/BIM, AI Agents, and 3D Immersive Review
- Three bullets: Produce faster with standardized outputs / Reduce manual errors and repetitive tasks / Enable the lead to visualize and modify without touching technical tools

### Slide 3 — Guiding Principle
- Core formula: Business Need → AI Orchestrator → AutoCAD/Revit → Quality Control → 3D Immersive Review → Decision → New Version
- Guiding principle: "AI prepares, humans decide, the lead visualizes and modifies."
- AI is not a replacement — it is a production accelerator and quality standardizer

### Slide 4 — Target Architecture Overview
- 5-layer stack visual:
  1. AutoCAD (2D Production)
  2. Revit / BIM (Structured Model)
  3. MCP (Model Context Protocol)
  4. 3D Immersive Review
  5. AI Orchestrator Agent

### Slide 5 — Layer 1 & 2: AutoCAD + Revit/BIM
- Layer 1 AutoCAD: Plans 2D, technical details, quick variants, execution drawings. Easiest starting point.
- Layer 2 Revit/BIM: Central BIM layer for intelligent objects (walls, doors, windows, levels, rooms, equipment, schedules)
- AI Scope: Generate drawing structures, manage layers, apply Roc4t standards, produce exports

### Slide 6 — Layer 3-5: MCP, 3D Review, AI Orchestrator
- Layer 3 MCP: Bridge between AI and design software. Options: Official Autodesk MCP / Open-source MCP / Custom Roc4t MVP MCP
- Layer 4 3D Immersive Review: Visualization and validation for non-technical stakeholders. Natural language change requests.
- Layer 5 AI Orchestrator: Planner → Generator → Verifier (multi-agent pattern). Reads natural language, chooses the right tool, launches actions, verifies errors, prepares exports.

### Slide 7 — Functional Workflow
- Step-by-step flow:
  1. Client/Lead expresses a need
  2. Designers prepare a first base
  3. AI automates repetitive tasks and prepares variants
  4. Project is consolidated into a clean model
  5. Model is pushed to 3D immersive viewer
  6. Lead explores the project
  7. Lead requests changes (e.g., "move this partition 20cm")
  8. AI applies modifications in Revit/AutoCAD
  9. 3D view is updated
  10. Project returns to validation or production

### Slide 8 — 30-Day Roadmap
- Phase 1: AutoCAD POC (Weeks 1-2) — Quick productivity gain on a narrow use case
- Phase 2: BIM / Revit Extension (Weeks 3-4) — Move from drawing to intelligent design automation
- Phase 3: 3D Immersive Review (Week 4+) — Make the project readable for the lead
- Phase 4: Industrialization (Month 2+) — Document standards, implement quality controls, stabilize the loop

### Slide 9 — Role of the Two Designers
- Phase 1 — Reference Production: Produce manually, serve as operational reference, document best practices
- Phase 2 — Assisted Production: Become operators of AI-assisted production, validate AI outputs, correct ambiguous cases
- Phase 3 — Supervision: Evolve into business supervisors, formalize know-how into exploitable rules
- Key insight: The AI must learn the Roc4t method, not invent its own.

### Slide 10 — Risk Matrix
- Top 5 risks table:
  - Scope too broad from start → High/High → Choose one simple use case
  - Standards poorly defined → High/High → Document templates, layers, naming before automation
  - AutoCAD/Revit linkage complicated → High/Medium → Validate each step separately
  - Too little human validation → High/Medium → Keep mandatory control workflow
  - Insufficient file security → High/Medium → Limit access and log actions
